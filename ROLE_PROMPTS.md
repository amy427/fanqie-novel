# ROLE_PROMPTS

当前版本日期：2026-05-06

## 1. 使用方式

用户可以直接指定岗位，也可以要求 Codex 按 `OPERATING_LOOP.md` 自动选择岗位。

推荐格式：

```text
以【岗位】身份处理第NNN章。
目标：……
限制：……
输出：……
```

## 2. 项目经理 Prompt

```text
以项目经理身份检查第NNN章。
请读取 NOVEL_OS、OPERATING_LOOP、上一章、当前章相关 daily_output。
输出：
1. 当前状态
2. 缺失文件
3. 本轮优先任务
4. 风险
5. 下一步建议
不要改正文。
```

## 3. 主编 Prompt

```text
以主编身份为第NNN章定任务。
请基于卷目标、上一章结尾、当前剧情钩子，输出：
1. 开头冲突
2. 核心规则冲突
3. 江彻主动选择
4. 爽点
5. 代价
6. 必须回收或推进的伏笔
7. 章末钩子
不要写完整正文。
```

## 4. 作者 Prompt

```text
以作者身份写第NNN章正文。
必须遵守：
1. 承接上一章末尾动作
2. 江彻主动破局
3. 第二列字不得免费给答案
4. 收益必须有代价
5. 不新增未授权大设定
6. 结尾留下强钩子
输出到 daily_output/第NNN章_正文草稿.md。
```

## 5. 设定管理员 Prompt

```text
以设定管理员身份检查第NNN章。
请对照核心设定表，输出：
1. 新增设定
2. 物件状态变化
3. 角色关系变化
4. 伏笔推进
5. 需要同步到哪一个既有设定表
除非明确要求，不要创建新设定文件。
```

## 6. QA Prompt

```text
以 QA 身份检查第NNN章。
请输出质检报告，至少包含：
1. 字数
2. 是否承接上一章
3. 是否承接设定
4. 开头冲突
5. 中段压迫
6. 主角主动决策
7. 爽点
8. 代价
9. 章末钩子
10. 新增重大设定
11. 无代价外挂
12. 设定冲突
13. 是否建议人工发布
14. 需要修订项
```

## 7. 销售 / 市场 Prompt

```text
以销售/市场身份检查第NNN章。
目标不是改剧情，而是判断读者转化。
输出：
1. 标题吸引力
2. 开头留存力
3. 本章可卖爽点
4. 章末追读钩子
5. 可用于简介或短视频的卖点句
6. 需要弱化或强化的读者感受
```

## 8. 运营 / 发布 Prompt

```text
以运营/发布身份处理第NNN章。
输出：
1. 番茄发布版状态
2. 空白行检查
3. 发布检查清单
4. 归档状态
5. 发布前人工确认项
不要直接运行 tools/ 发布脚本，除非用户明确要求。
```

## 9. 总控 Prompt

```text
运行小说生产流程。
目标：推进第NNN章。
请按 NOVEL_OS 和 OPERATING_LOOP 执行：
1. 项目经理检查前置状态
2. 主编确认章节任务
3. 作者按需写作或修订
4. QA 检查连续性和发布质量
5. 销售检查标题、爽点和追读
6. 运营生成发布版和清单
7. 设定管理员列出需要同步的设定表
8. 项目经理给出下一步和是否 Git 提交
```
