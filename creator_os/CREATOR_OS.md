# CREATOR_OS

当前版本日期：2026-05-06

## 1. 目标

让 Codex 不只服务单本小说，而是服务“冷痕”这个创作者的长期内容生产、复盘和资产积累。

## 2. 层级

1. 项目层：单本小说的设定、章节、QA、发布。
2. 产品层：书名、简介、标题、标签、读者定位、追读表现。
3. 内容资产层：可复用爽点、规则怪谈模板、标题库、简介库。
4. 个人知识层：作者偏好、写作原则、弱点和成长记录。
5. 商业运营层：平台表现、收益、推广、粉丝反馈。

## 3. 当前主项目

`fanqie-novel`：`高武：违规者才配活着`

当前优先级：

1. 稳定旧河公寓后半段收束。
2. 第030章承接上一任J-317和第七节车厢。
3. 发布后回填真实读者反馈和章节数据。

## 4. 使用方式

当用户说“按最顶级做法”时，Codex 应优先判断问题属于：

1. 单次任务
2. 小说项目操作系统
3. 自动化闭环
4. 读者反馈和商业复盘
5. 创作者长期资产

然后先补系统缺口，再执行具体文件操作。
